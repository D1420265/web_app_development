# 產品需求文件 (PRD) - 食譜管理系統

## 1. 專案概述
- **背景與動機**：日常想要記錄自己喜歡或者是常做的料理往往散落各處。有時打開冰箱不知道要做什麼菜，因此需要一個可以統一儲存食譜，並且能依照現有的食材進行料理推薦的系統。
- **目標用戶**：我（個人使用者）。
- **核心價值主張**：提供一個簡單好用的個人化食譜整理庫，不僅能永久保存喜愛的料理作法，還能在不知所措時透過「食材推薦」功能解決日常煮飯的煩惱。

## 2. 功能需求
系統將提供以下主要功能（使用者故事）：

1. **新增食譜**
   - 作為使用者，我希望能夠新增食譜（包含標題、所需食材、料理步驟等），以便將好用的食譜保存下來。
2. **搜尋食譜**
   - 作為使用者，我希望可以透過關鍵字搜尋食譜的名稱或內容，以便快速找到我想做的那道菜。
3. **用食材推薦食譜**
   - 作為使用者，我希望可以輸入目前手邊（例如冰箱裡）現有的食材，系統可以推薦包含這些食材的食譜，以便有效利用食材並減少浪費。
4. **編輯與刪除食譜**
   - 作為使用者，我希望能夠修改已存在的食譜內容或刪除不再需要的食譜，以便維護食譜庫的實用性與整潔。
5. **檢視食譜詳細內容**
   - 作為使用者，我希望點擊食譜清單中的某項後，可以看到完整的所需材料與條列式的料理步驟，以便我能在廚房一邊看一邊料理。

## 3. 非功能需求
- **技術限制**：後端採用 Flask 框架、模板引擎使用 Jinja2、資料庫使用簡單輕量的 SQLite 儲存。
- **效能考量**：這是一個輕量化的個人系統，需確保新增食譜與搜尋功能（包含推薦）的回應時間在 1 秒以內即可，以保持良好且流暢的操作體驗。
- **安全考量**：包含基礎防護如避開 SQL Injection 與簡單的錯誤提示。若未來要部屬在外網，可以加入簡易的使用者身分驗證（Login）防護。

## 4. MVP 範圍
- **Must Have（必須有）**
  - 食譜的 CRUD（新增、查詢條列與詳細內容、修改、刪除）。
- **Should Have（應該有）**
  - 使用關鍵字搜尋食譜（名稱）。
  - 使用「食材」作為關鍵字來推薦相關食譜。
- **Nice to Have（最好有）**
  - 幫食譜加上圖片或標籤分類（例如：甜點、湯品、主食）。
  - 自動換算食材份量。
